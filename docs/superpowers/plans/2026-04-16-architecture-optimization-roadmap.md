### Phase 3: Execution Robustness (2 commits)

- **B2 — Adaptive Budget**: `budget_flex` config on stages. Auto-approve extensions when agent outputs `_needs_extension: true`. Hard ceiling via `max_extensions` (1-5).

## What Remains

### A1: Phased Pipeline (Phase 4 — the final architectural piece)

**Problem:** Generator AI predicts all stages at t=0 from one sentence. Complex pipelines (15-27 stages) degrade because the plan becomes stale by stage 10.

**Solution:** Meta-pipeline orchestrates phases via sub-pipeline calls. Each phase's internal stages are generated by a planning-stage that reads actual store data from prior phases.

**Technical approach:** Extend `PipelineCallRuntimeConfig` with `pipeline_source: "store"` to read inline pipeline definitions from the store. Each phase is an independent sub-task with its own XState machine.

**Required work:**
1. Type + schema: add `pipeline_source` and `pipeline_key` to PipelineCallRuntimeConfig
2. Modify `pipeline-executor.ts` to load pipeline from store
3. Validate store-sourced pipeline definitions (same schema + logical validation)
4. Create planning-stage prompt template
5. Proof-of-concept: convert tech-research to phased model
6. Test retry/resume/persistence across phases

**Estimated effort:** 2-3 weeks including design spike

**Prerequisites:** A3 (store schema) ✅, A2 (scratch pad) ✅ — both complete.
