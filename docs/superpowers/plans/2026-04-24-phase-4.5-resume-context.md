## 10. Gotchas discovered along the way

1. **cwd drifts** between repo root and `apps/server` across Bash calls. Always
   `pwd` before running `npx tsc` etc.
2. **tsc 错误提示 "This is not the tsc command you are looking for"** 意味着
   cwd 不对。一定要 `cd apps/server` 先。
3. **FK violation stderr during real-executor test** is expected (test
   intentionally violates FK to verify fallback to NoopWriter).
4. **Flaky test** occasionally (1498/1498 pass then next run 1497/1498 with
   1 stderr failure) — ran twice last time, second pass clean. If a
   seemingly random test fails once in full suite, re-run before investigating.
5. **`npx vitest run` in repo root uses v4.1.5; in apps/server uses v4.0.18**
   — two different vitest installs. Both work.
6. **TaskRegistry termination promise contract**：unregister() 必须在
   signalTermination() 之后调用（或 unregister 自己 defaults to kind=natural
   if no prior signal）。
7. **`interruptWaitMsOverride` 为 test 而生**：生产调用链中默认 30s；test
   注入短值（50-2000ms）避免超 vitest 10s default timeout。
8. **CLAUDE.md "Step 0 rule"**：大文件重构前先删死代码独立 commit。我在
   Stage 5B Task 8 部分遵守（245-line retired A8 body 直接删除而不是重构）。
9. **kernel-next 现在是唯一引擎**：不要提议重新引入 Gemini/Codex/legacy
   engine。不要碰已删的 `apps/server/src/machine/` / `edge/` / `agent/`
   目录（全部退役）。
10. **Pipeline YAML 由 AI 写，用户不手写**：设计时不要考虑"用户阅读友好"
    的 DSL 语法优化 —— 目标读者是 AI agent。
