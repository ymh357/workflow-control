### Task 28 вҖ” multi-session token optimization (612921e)

- **Prior behaviour bug**: `buildSystemPromptAppend` sliced every port
  value JSON to 400 chars with silent `[...]`. Agents downstream of
  large producers read the first 400 chars as if it were the full
  value вҖ” **correctness bug, not just token waste**.
- New behaviour: if `JSON.stringify(value).length вүӨ 1024` inline the
  full value; otherwise emit a summary line with type + byte count +
  explicit `mcp__kernel_next__read_port` instruction. `read_port` was
  already in the agent's `combined` MCP surface.
- Also exported `buildSystemPromptAppend` + `exampleValueFor` from
  real-executor.ts so tests use the real implementation (previous
  test file had a stale local copy).
- 5 new tests in `real-executor.empty-inputs.test.ts` (10 total),
  covering inline small, exact-threshold boundary, large-string
  summary, large-object type label, mixed.
- **SDK prompt caching was investigated**: Claude Agent SDK 0.2.63
  exposes `systemPrompt` as `{ preset, append: string }` only. No
  `cache_control` content-block API. Explicit caching wiring is not
  possible at this SDK version; relying on whatever caching the
  Claude Code preset applies internally. **No cache wiring landed.**

### Task 29 вҖ” B9 partial (3199695)

- **Roadmap B9 has two halves**: (a) `git reset --hard before_sha` on
  worktree, (b) inject superseded attempt's diff into successor
  attempt as reference. **Only (b) shipped**. (a) requires task-worktree
  ownership semantics (which task owns which workdir) that kernel-next
  does not yet have вҖ” resetting a shared workdir on migration would
  clobber other tasks. Full B9 is gated on Phase 5C worktree lifecycle.
- New table `migration_hints` (hint_id PK, task_id, stage_name,
  from/to_version, previous_attempt_id, previous_diff_text, bytes,
  note, created_at, consumed_at). Partial index on (task_id,
  stage_name) WHERE consumed_at IS NULL for O(1) executor lookup.
- New module `apps/server/src/kernel-next/hot-update/migration-hints.ts`:
  `writeMigrationHint` / `peekUnconsumedHint` / `consumeHint`
  (take-once via `UPDATE ... RETURNING` guarded by `consumed_at IS NULL`).
- `migration-orchestrator.ts`: after supersede TX commits + before
  startRunner fires, reads rerunFrom stage's most recent superseded
  attempt, looks up its stage_checkpoint diff, writes a migration_hint
  row. Graceful degradation when checkpoint missing or diff too large.
- `real-executor.ts`: `buildSystemPromptAppend` now accepts optional
  `MigrationHint` and renders `### Migration note (B9)` block with
  version transition + note + diff fenced in ` ```diff ` (truncated at
  8000 chars in-prompt; full diff remains in `stage_checkpoints`).
  `doAttempt` consumes the hint atomically per attempt so stage retries
  inside one run don't re-inject.
- Tests: 12 hint unit + 7 real-executor migration-hint + 2 orchestrator
  integration + 3 schema = 24 new cases.
- Roadmap B9 row updated to вҡ пёҸ partial.
