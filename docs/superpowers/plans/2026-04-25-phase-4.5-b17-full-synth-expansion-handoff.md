## 2. Per-task landing

### T1 — Schema: stage_attempts.fanout_element_idx
SHA: `36393b8`

- New INTEGER NULL column on `stage_attempts` via KERNEL_NEXT_SCHEMA.
- CHECK constraint: `kind != 'fanout_element' OR fanout_element_idx IS NOT NULL`
  — per-element rows MUST carry an idx; other kinds are free to be NULL.
- Partial index `idx_sa_fanout_success` on
  `(task_id, stage_name, fanout_element_idx) WHERE kind='fanout_element' AND status='success'`
  — supports the T3 hot-path "which indices already succeeded for
  this (task, stage)?".
- `initKernelNextSchema` gained a zero-historical-compat drop+rebuild
  trigger: if `stage_attempts` exists but lacks `fanout_element_idx`,
  every kernel-next table is dropped and KERNEL_NEXT_SCHEMA runs
  fresh. Aligns with CLAUDE.md §Retired areas.
- 5 unit tests cover column presence, NULL default, CHECK rejection,
  permit NULL on other kinds, permit idx on other kinds.
- Two existing test fixtures that `INSERT` directly into
  `stage_attempts` (migration-orchestrator.fanout.test,
  compare-runs.test) updated to supply the new column for
  `fanout_element` rows.

### T2 — PortRuntime + executors thread fanoutElementIdx
SHA: `7e2e45b`

- `StartAttemptArgs.fanoutElementIdx?: number` + `PortRuntime.startAttempt`

### T3 — orchestrateFanoutStage index-skip
SHA: `5ad573e`

- Before the element loop, query `stage_attempts` for
  `(task_id, stage_name, kind='fanout_element', status='success',
   fanout_element_idx < sourceValue.length)` rows. Build a
  `Map<number, Record<string,unknown>>` from idx → declared output
  port values (fetched from port_values).
- In the loop, if `preservedByIdx.has(i)`, push its outputs into the
  aggregate and `continue` — no new attempt, no executor call.
- Otherwise, existing path: `executor.executeStage({...,
  fanoutElementIdx: i})`, collect outputs, push.
- Out-of-range guard: the SQL's `fanout_element_idx < sourceValue.length`
  predicate ignores preserved attempts from a longer previous source
  array (e.g. after an upstream filter change).
- 4 unit tests: partial skip (2 of 4 indices preserved), fresh run
  (no prior work), full skip (all preserved, handler never called),
  shrunk source array.

### Synth T1 — safe-range expansion to subAgents
SHA: `9c62608`

- Motivation: prior T6 iteration restricted `configPatch` to
  `{promptRef}` only. `AgentStageSchema.config` declares two keys:
  `promptRef` and `subAgents`. Nothing else is a legitimate
  AgentStage config field in the current IR. The previous "reads /
  writes / budget" language in the old handoff was mis-named —
  those fields don't exist in the schema.
- `claude-sdk-patch-synthesizer.ts`:
  - `allowedKeys = ["promptRef", "subAgents"]`.
  - Every subAgents entry runs through `SubAgentDefSchema.safeParse`
    (the same Zod validator the IR submit path uses). A single
    invalid entry rejects the whole patch.
  - SYSTEM_PROMPT updated: documents the new shape, lists
    subAgents field constraints (name regex, non-empty description/
    prompt, optional tools/model/maxTurns, empty array meaning "drop
    all subagents"). Adds hard constraints 3 + 4 governing the
    configPatch keyset and subAgents structural validity.
- `propose-pipeline-fix.ts` `isSafeRangePatch`: shared
  `SAFE_CONFIG_KEYS = {promptRef, subAgents}` Set, enforced at the
  wrapper layer too. Defense in depth — rogue patches that bypass
  the synthesiser parser still get blocked before reaching the
  caller.
- 6 new synth unit tests (valid subAgents, combined with promptRef,
  invalid name, missing prompt, still reject other keys, empty
  array ok) + 2 new wrapper tests (reject budget key, accept
  subAgents) + 1 existing synth test adjusted (now uses `budget`
  as the canonical rejected key since subAgents moved into
  allowedKeys).

## 4. Invariants added this session (don't break)

16. **fanout_element rows MUST have fanout_element_idx**. Schema
    CHECK makes this unforgeable. Any new writer path (e.g. a
    future debug tool that inserts fanout_element rows directly)
    must supply it. Only PortRuntime.startAttempt (via the mock /
    script / real executors) is a legitimate writer today.

17. **Preserved fanout_element attempts are READ-ONLY after
    migration.** `orchestrateFanoutStage` treats them as frozen
    historical outputs — it reads port_values and reuses the
    values, but does not touch the attempt row or the port_values
    rows themselves. This is consistent with B17 T2's supersede
    carve-out.

18. **AgentStage configPatch safe-range tracks schema.ts exactly.**
    Any new field added to `AgentStageSchema.config` must get
    mirrored into both
    `claude-sdk-patch-synthesizer.ts:allowedKeys` and
    `propose-pipeline-fix.ts:SAFE_CONFIG_KEYS`. The reverse is
    also true — a key removed from schema must be dropped from
    both. Without a schema entry, the AI has no structure to
    synthesise against.
